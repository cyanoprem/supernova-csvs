## Guiding Principle

The 3rd-pay data by "need" tells us exactly what retains:

| Need | 3rd Pay % (of 2nd-pay) | Retention Signal |
|---|---|---|
| English doubts and translations | 55.9% | **Daily ongoing need** |
| Grammar correction in emails/docs | 52.4% | **Daily ongoing need** |
| Improve spoken English | 54.0% | **Daily ongoing need** |
| Workplace communication | 53.1% | **Daily ongoing need** |
| Interviews | 28.9% | Event-based, churns after |
| Confidence | 24.7% | Vague, no milestone |
| Govt exams | 21.5% | Event-based, churns after |

**Content that serves a daily, repeating work need retains. Content that serves a one-time event doesn't.** Every new scenario should pass the test: "Will this user need to do this AGAIN tomorrow?"

## NEW MULTI-QUESTION (STRUCTURED) SCENARIOS TO BUILD

### Tier 1: Daily-use workplace scenarios (highest 3rd-pay impact)

These map to the situations users bring into free-form 1000+ times each. They pass the "will they need this again tomorrow?" test.

**MQ-1: "Write a Message to Your Boss"**
- Format: User hears a Hindi workplace situation -> constructs an English message -> gets corrections
- Levels:
  - A1: Simple status updates ("The work is complete", "I will be late today")
  - A2: Detailed updates ("The welding work on tank 3 is 70% complete. Fitting will start tomorrow.")
- Evidence: 218 free-form messages of users drafting daily reports; user 020b8d29 does this every evening
- 3rd-pay lever: Serves a DAILY need -- users send work updates every day

**MQ-2: "Ask for Permission or Leave"**
- Format: Roleplay scenarios where user practices asking boss for leave, arriving late, taking time off
- Situations:
  - "You have a fever and can't come to work. Message your supervisor."
  - "You need to leave early for a family event. Ask your manager."
  - "You were late today. Explain to your boss why."
- Evidence: 217 leave/permission messages in free-form; qual user Narendra needs this for school meetings
- 3rd-pay lever: Recurring workplace situation

**MQ-3: "Give a Work Update on a Phone Call"**
- Format: Simulated phone call -- AI speaks as the boss/client, user responds
- Situations:
  - Boss asks: "What's the status?" -> User gives update
  - Client calls: "When will the delivery arrive?" -> User responds
  - Colleague asks: "Can you help with X?" -> User accepts/declines
- Evidence: 911 phone-call-script messages in free-form; 6,275 total phone/call-related messages
- 3rd-pay lever: Phone calls happen multiple times per day at work

**MQ-4: "Apologize and Explain a Problem"**
- Format: User practices explaining delays, mistakes, issues to boss/client
- Situations:
  - "The delivery is delayed by 2 days. Inform the client."
  - "You made an error in the report. Explain what happened."
  - "A machine broke down. Update your supervisor."
- Evidence: 1,214 apology/excuse messages; qual quote: "If I have made a mistake in my work or time, what message should I give?"
- 3rd-pay lever: Mistakes happen daily; knowing how to explain them in English is ongoing

**MQ-5: "Describe Your Work to Someone New"**
- Format: Self-introduction in a workplace context -- not personal ("My name is...") but professional ("I work as... I handle...")
- Situations:
  - "A new colleague asks what you do. Explain your role."
  - "A client visits your workplace. Introduce your team's work."
  - "Your boss asks you to explain your department to a visitor."
- Evidence: 888 self-introduction messages; qual users Saiba, Savita, Deb all need this
- 3rd-pay lever: Builds confidence for the most common workplace interaction

### Tier 2: Weekly-use workplace scenarios (moderate 3rd-pay impact)

**MQ-6: "Hindi Thought -> English Sentence" Production Drills**
- Format: User sees a Hindi sentence on screen (or hears it) -> must SPEAK the English translation -> gets correction and model answer
- This is the exact "Think in Hindi, Speak in English" drill that 5 out of 17 qual users requested
- Sentences should be WORKPLACE sentences, not textbook:
  - A1: "Mera kaam ho gaya hai" -> "My work is done"
  - A1: "Sir, main kal nahi aa paunga" -> "Sir, I won't be able to come tomorrow"
  - A2: "Delivery mein do din ki deri hogi" -> "The delivery will be delayed by two days"
  - A2: "Kya aap yeh report check kar sakte hain?" -> "Can you please check this report?"
- Evidence: Gungun (126 activities, "main connect nahi kar paati sentences ko"); Savita ("bolte samay thoda atak"); the core production gap
- 3rd-pay lever: Directly addresses the #2 churn problem -- users who complete activities but can't speak

**MQ-7: "Understand What Your Boss Said"**
- Format: AI says a common English workplace instruction -> user must understand and respond correctly
- Situations:
  - "Please send me the report by 5 PM" -> User confirms
  - "Can you coordinate with the other team on this?" -> User asks clarifying questions
  - "I need you to handle the client call tomorrow" -> User responds appropriately
- Evidence: Akash (B.Com student, "questions ko understand karne mein galti hoti thi"); comprehension-in-context gap
- 3rd-pay lever: Daily need -- understanding English instructions at work

**MQ-8: "Numbers, Dates, and Amounts in English"**
- Format: Practice saying quantities, dates, money amounts, measurements in workplace context
- Situations:
  - "The payment is 77,000 rupees" -> correct pronunciation and phrasing
  - "The meeting is on 15th January at 3 PM"
  - "We need 200 kg of material"
  - "The project is 70% complete"
- Evidence: 339 payment/invoice messages; user 020b8d29 constantly deals with material quantities and amounts
- 3rd-pay lever: Blue-collar/supervisor users deal with numbers daily

**MQ-9: "Email and WhatsApp Phrases for Work"**
- Format: Learn and practice common email/message openers, closers, and body phrases
- Phrases to drill:
  - "Please find attached...", "As discussed...", "I am writing to inform you..."
  - "Could you please...", "I would like to request...", "Thank you for your support"
  - "Dear Sir/Madam", "Best regards", "Looking forward to your reply"
- Evidence: 803 "email" keyword messages; 3,064 email scenario messages; user quote: "yeh draft meri office ko bhejna hai thoda best professional aisa proper lagna chahiye"
- 3rd-pay lever: Users send work messages daily and want to sound professional

**MQ-10: "Respond to Common Workplace Questions"**
- Format: Quick-fire Q&A where AI asks common workplace questions and user practices natural responses
- Questions:
  - "How's the project going?"
  - "When will this be ready?"
  - "Did you finish the task I assigned?"
  - "Can you work overtime today?"
  - "What happened with the delivery?"
- Evidence: 2,874 boss-interaction messages; 2,069 team-communication messages
- 3rd-pay lever: These questions come up every single day

## NEW ASK AI (FREE-FORM) SCENARIOS TO BUILD

### Tier 1: Replace generic with workplace-specific

**AI-1: "Talk to Your Boss" (Roleplay)**
- Nova plays the role of the user's boss/supervisor
- User practices: giving updates, asking for leave, explaining problems, responding to feedback
- The AI should be slightly formal and occasionally push back ("Why is this delayed?") to simulate real pressure
- Differentiate from MQ: this is unscripted conversation, not fill-in-the-blank
- Evidence: 2,874 boss-related messages; the single most common workplace interaction
- **This is the #1 missing Ask AI scenario.**

**AI-2: "Draft My Work Message" (Upgrade existing "Write an Email/Message")**
- Current "Write an Email/Message" is generic. Split it into workplace-specific variants:
  - "Draft a Status Update" -- user describes what happened, AI creates professional message
  - "Draft a Leave Request" -- user says why they need leave, AI formats it
  - "Draft a Complaint/Escalation" -- user describes the problem, AI writes formal complaint
  - "Draft a Payment/Invoice Message" -- user gives amounts, AI creates professional message
- After drafting: AI teaches 3 key phrases from the draft ("Today you learned: 'Please find below...', 'I would like to bring to your attention...', 'Kindly approve...'")
- Evidence: 7,023 existing email/message msgs; 17,938 translate-to-English msgs used for this purpose
- **This bridges "tool use" and "learning" -- the user gets their work done AND learns phrases.**

**AI-3: "Practice a Client/Customer Call"**
- Nova plays a client or customer
- Scenarios:
  - Client asks about product/service details
  - Customer complains about a delay
  - Client asks for a price quote
  - Customer wants to reschedule
- Evidence: 1,090 client-handling messages; qual users Saiba (makeup clients), Dineshbabu (IT clients)

**AI-4: "Mock Job Interview"**
- Nova plays interviewer
- Starts simple: "Tell me about yourself" -> "What is your experience?" -> "Why should we hire you?"
- AI gives feedback after each answer on grammar + content + confidence
- Evidence: 80 interview-prep messages in free-form; 124 "interview" keyword mentions; 5 out of 17 qual users requested this; interview prep has 28.9% 3rd-pay conversion (decent for event-based)
- Note: While event-based needs have lower 3rd-pay, interview prep is the #1 requested missing scenario in qual. It's a acquisition + conversion driver even if not a long-term retention driver.

**AI-5: "Correct My English" (Upgrade existing "Correct my Grammar")**
- User speaks or types any English sentence -> AI corrects it AND explains why
- Specifically: show the wrong version, the corrected version, and the grammar rule
- Current "Correct my Grammar" exists but only has 1,094 messages -- low adoption, probably poor discoverability
- Evidence: Dineshbabu (qual): "real time conversation andha conversation irukkum bodhu adha andha idathula correct pannalai na ennodha mistake theriyadhu"; Savita: "kuch main galat ho toh mere ko woh correct kar de"
- 3rd-pay lever: Ongoing daily need

**AI-6: "Explain in Simple English" (Comprehension Helper)**
- User pastes/speaks any English text they received (from boss, email, document, WhatsApp)
- AI explains it in simple English + Hindi
- This is the reverse of "Translate to English" -- user RECEIVED English they don't understand
- Evidence: Akash: "questions ko understand karne mein thoda galti hoti thi"; Divyani confused by app questions
- 3rd-pay lever: Users receive English messages at work daily

### Tier 2: Occupation-specific conversation scenarios

**AI-7: "English for [Your Job]" (Dynamic Occupation Roleplay)**
- Uses the user's occupation from onboarding/memory to create job-specific conversations
- Construction supervisor: site inspection conversations, material order discussions
- Shop manager: customer interactions, inventory discussions, supplier calls
- Teacher: parent-teacher meetings, classroom instructions, colleague discussions
- Sales: cold calls, product pitching, objection handling
- Accountant: explaining financial statements, discussing payments, audit conversations
- Evidence: Every Hindi qual user had a different profession; 8/17 cited generic content as #1 pain point
- Implementation: This doesn't require building 20 separate scenarios. One Ask AI scenario that reads `user.occupation` and adapts the roleplay context dynamically.

## WHAT TO DEPRIORITIZE/REDUCE IN CURRENT CURRICULUM

The current MQ curriculum is heavily skewed toward tense revision. The top 10 MQ scenarios by volume are ALL grammar drills. These shouldn't be removed (grammar-focused users at 6% of power users still need them, and "grammar improvement" is a top positive signal in qual), but their share of the first 30 days should drop from ~60% to ~30% for Work users, replaced by workplace scenarios.

**Current state for a Hindi A1 Work user's first 30 days:**
1. Conversation Practice (onboarding)
2. Revise Simple Present Tense
3. Revise Present Continuous Tense
4. Revise Articles
5. Revise Adjectives
6. ... more grammar

**Target state for a Hindi A1 Work user's first 30 days:**
1. Conversation Practice (onboarding -- detect occupation)
2. Write a Message to Your Boss (MQ-1) -- immediately workplace-relevant
3. Revise Simple Present Tense -- grammar foundation
4. Hindi -> English Production Drill (MQ-6) -- workplace sentences
5. Ask for Permission or Leave (MQ-2) -- practical scenario
6. Revise Present Continuous Tense -- grammar
7. Give a Work Update on a Phone Call (MQ-3) -- workplace
8. Email/WhatsApp Phrases for Work (MQ-9) -- practical
9. Revise Adjectives -- grammar
10. Apologize and Explain a Problem (MQ-4) -- workplace

**Alternating pattern: workplace -> grammar -> workplace -> grammar.** This way the user sees value (workplace relevance) interleaved with foundation (grammar). By day 30, they've both learned past tense AND practiced writing a report to their boss.

**This is config-only, no code changes.** The scenarios themselves need to be authored in the content system, but the routing is free.

For Ask AI scenarios: update `getSystemUserOccupationVars()` to pass rich occupation + need data to the LLM prompts (30-min code fix). This makes ALL existing Ask AI conversations workplace-aware immediately, even before new dedicated scenarios are built.
