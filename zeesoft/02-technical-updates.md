Concept: Domain-Specific Communication - Technical Updates
Level: A1
Lesson type: Can Do

Scenarios to use for sentences: Sprint standup update, Bug report to the team lead, Deployment status update, System outage communication, Feature release announcement, Code review feedback, Migration progress report

---

Sub-concept 1: Reporting current status
Description: The learner can share what they are working on and its current state
Sentence structures:
1. "I'm working on [task]. It's [status]."
   Example: "I'm working on the login page. It's almost done."
2. "The [system] is [state] right now."
   Example: "The staging server is down right now."
3. "[Item] is at [value]. We're [on track / a bit behind]."
   Example: "Test coverage is at 72%. We're a bit behind."

---

Sub-concept 2: Explaining blockers
Description: The learner can explain what is stopping their work and what help they need
Sentence structures:
1. "I'm stuck on [task] because [reason]."
   Example: "I'm stuck on the API work because the docs aren't ready yet."
2. "This depends on [task], which should be done by [day]."
   Example: "This depends on the database update, which should be done by Thursday."
3. "We found [issue] in [task], so we [action taken]."
   Example: "We found a bug in the build setup, so we rolled back for now."

---

Sub-concept 3: Suggesting solutions
Description: The learner can propose an approach and give a time estimate
Sentence structures:
1. "I think we should [approach]. It'll take about [time]."
   Example: "I think we should use a separate service for notifications. It'll take about two weeks."
2. "If we [action], we can finish by [date]."
   Example: "If we split the work, we can finish by end of March."
3. "We can either [option A] or [option B]."
   Example: "We can either fix the current module or build a new one."
